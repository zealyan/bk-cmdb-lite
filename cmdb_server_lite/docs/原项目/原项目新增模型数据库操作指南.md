# CMDB新增模型数据库操作指南

## 概述

本文档说明在CMDB中新增自定义模型（如：应用系统、应用子系统）时，需要创建的数据库表及其依赖关系。

## 案例场景

以 **"应用系统"** 和 **"应用子系统"** 为例：
- 一个 **应用系统** 可以关联 **N个应用子系统**
- 关联关系：1:N（一对多）

## 数据库表依赖关系图

```
┌─────────────────────────────────────────────────────────────────┐
│                        新增模型流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. cc_ObjClassification (模型分组表)                            │
│     ↓                                                           │
│  2. cc_ObjDes (模型定义表)                                       │
│     ↓                                                           │
│  3. cc_PropertyGroup (属性分组表)                                │
│     ↓                                                           │
│  4. cc_ObjAttDes (模型属性表)                                     │
│     ↓                                                           │
│  5. cc_AsstDes (关联类型定义表)                                   │
│     ↓                                                           │
│  6. cc_ObjAsst (模型关联关系表)                                   │
│     ↓                                                           │
│  7. cc_ObjectBase (实例数据表)                                    │
│     ↓                                                           │
│  8. cc_InstAsst (实例关联表)                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 数据库表详细说明

### 1. cc_ObjClassification（模型分组表）

**作用**：存放模型分组信息，用于在前端界面上对模型进行分类展示。

**表结构**：

| 字段 | 类型 | 描述 |
|------|------|------|
| _id | ObjectId | 数据唯一ID |
| bk_classification_id | String | 分组唯一标识 |
| bk_classification_name | String | 分组名称 |
| bk_classification_type | String | 分组类型 |
| bk_classification_icon | String | 分组图标 |
| id | NumberLong | 自增ID |
| bk_supplier_account | String | 开发商ID（固定值：0） |

**示例数据**：

```sql
-- 应用系统分组
INSERT INTO cc_ObjClassification (
    bk_classification_id,
    bk_classification_name,
    bk_classification_type,
    bk_classification_icon,
    bk_supplier_account
) VALUES (
    'application_system',
    '应用系统',
    'custom',
    'icon-cc-application',
    '0'
);
```

### 6. cc_ObjAsst（模型关联关系表）

**作用**：定义具体模型之间的关联关系。

**表结构**：

| 字段 | 类型 | 描述 |
|------|------|------|
| _id | ObjectId | 数据唯一ID |
| id | NumberLong | 自增ID |
| bk_obj_id | String | 源模型ID |
| bk_asst_obj_id | String | 目标模型ID |
| bk_asst_id | String | 关联类型ID |
| bk_obj_asst_id | String | 关联关系唯一标识 |
| bk_obj_asst_name | String | 关联关系名称 |
| mapping | String | 对应约束：n:1/1:n/n:n |
| on_delete | String | 删除时行为：cascade/none |
| ispre | Boolean | 是否系统预置 |
| bk_supplier_account | String | 开发商ID |

**示例数据**：

```sql
-- 应用系统 包含 应用子系统 (1:N)
INSERT INTO cc_ObjAsst (
    bk_obj_id,
    bk_asst_obj_id,
    bk_asst_id,
    bk_obj_asst_id,
    bk_obj_asst_name,
    mapping,
    on_delete,
    ispre,
    bk_supplier_account
) VALUES (
    'bk_application_system',
    'bk_application_subsystem',
    'application_contain',
    'system_contain_subsystem',
    '应用系统包含子系统',
    '1:n',
    'cascade',
    false,
    '0'
);
```

**关联关系说明**：

| mapping值 | 含义 |
|-----------|------|
| 1:1 | 一对一 |
| 1:n | 一对多 |
| n:1 | 多对一（等价于1:n，从另一角度看） |
| n:n | 多对多 |

**on_delete选项**：

| 值 | 含义 |
|----|------|
| cascade | 级联删除 |
| none | 不删除，仅解除关联 |

## 完整创建步骤总结

### 步骤1：创建模型分组（可选）

```sql
INSERT INTO cc_ObjClassification (...) VALUES (...);
```

### 步骤2：创建模型定义

```sql
INSERT INTO cc_ObjDes (...) VALUES (...);
```

### 步骤3：创建属性分组

```sql
INSERT INTO cc_PropertyGroup (...) VALUES (...);
```

### 步骤4：创建模型属性

```sql
INSERT INTO cc_ObjAttDes (...) VALUES (...);
```

### 步骤5：创建关联类型定义（如需要）

```sql
INSERT INTO cc_AsstDes (...) VALUES (...);
```

```sql
INSERT INTO cc_ObjAsst (...) VALUES (...);
```

### 步骤7：创建实例数据表

CMDB系统会自动创建实例数据表，无需手动创建。

### 步骤8：创建实例关联数据

## 注意事项

1. **表名格式**：实例表和实例关联表的表名需要包含 `bk_supplier_account` 和 `bk_obj_id`

2. **字段类型**：确保属性字段的类型与业务需求匹配

3. **必填字段**：设置 `isrequired=true` 的字段在创建实例时必须提供

4. **关联约束**：根据业务需求选择正确的 `mapping` 值和 `on_delete` 行为

5. **级联删除**：`on_delete=cascade` 时，删除主实例会同时删除关联实例

## 常见问题

### Q1: 实例表为什么需要特殊的命名格式？

A1: CMDB使用 `bk_supplier_account` 来区分不同开发商的数据，使用 `pub` 表示公共实例（不受业务范围限制）。

### Q2: 如何创建多对多关联？

A2: 在 `cc_ObjAsst` 中设置 `mapping='n:n'`，系统会自动处理关联关系。

### Q3: 如何查看已创建的模型？

A3: 查询 `cc_ObjDes` 表可以获取所有已创建的模型定义。
