# CMDB新增模型数据库操作指南

## 概述

本文档说明在CMDB中新增自定义模型（如：应用系统、应用子系统）时，需要创建的数据库表及其依赖关系。

**文档版本说明**：
- 本文档基于 cmdb_server_lite 项目（使用 DuckDB 数据库）编写
- 原项目使用 MongoDB，Lite 版本使用 DuckDB，但**保留了原项目的表名结构**
- 本文档同时提供原项目（MongoDB）和 Lite 版本（DuckDB）的说明

## 案例场景

以 **"应用系统"** 和 **"应用子系统"** 为例：
- 一个 **应用系统** 可以关联 **N个应用子系统**
- 关联关系：1:N（一对多）

### 1. cc_ObjDes（模型定义表）

**对应原项目表**：`cc_ObjDes`（完全相同）

**作用**：存放模型的基本定义信息

**表结构**（来自 `/workspace/bk-cmdb/cmdb_server_lite/migrate_data.py` 第27-46行）：

```python
db.execute("""
    CREATE TABLE cc_ObjDes (
        _id VARCHAR,
        id INTEGER,
        bk_obj_id VARCHAR PRIMARY KEY,
        bk_obj_name VARCHAR,
        bk_obj_icon VARCHAR,
        bk_obj_type VARCHAR DEFAULT 'mainline',
        bk_classification_id VARCHAR,
        bk_ispaused BOOLEAN DEFAULT FALSE,
        bk_ishidden BOOLEAN DEFAULT FALSE,
        obj_sort_number INTEGER DEFAULT 0,
        description VARCHAR,
        creator VARCHAR DEFAULT 'admin',
        modifier VARCHAR DEFAULT 'admin',
        create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        last_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        bk_supplier_account VARCHAR DEFAULT '0'
    )
""")
```

**表字段说明**：

| 字段 | 类型 | 描述 |
|------|------|------|
| _id | VARCHAR | 文档ID |
| id | INTEGER | 记录ID |
| bk_obj_id | VARCHAR | 模型唯一标识（主键） |
| bk_obj_name | VARCHAR | 模型名称 |
| bk_obj_icon | VARCHAR | 模型图标 |
| bk_obj_type | VARCHAR | 模型类型（mainline/association） |
| bk_classification_id | VARCHAR | 模型所属分类 |
| bk_ispaused | BOOLEAN | 是否暂停 |
| bk_ishidden | BOOLEAN | 是否隐藏 |
| obj_sort_number | INTEGER | 排序号 |
| description | VARCHAR | 描述信息 |
| creator | VARCHAR | 创建人 |
| modifier | VARCHAR | 修改人 |
| create_time | TIMESTAMP | 创建时间 |
| last_time | TIMESTAMP | 更新时间 |
| bk_supplier_account | VARCHAR | 供应商账号 |

**示例数据**：


## API接口支持

Lite 项目通过 FastAPI 提供以下接口来操作模型数据：

### 模型相关接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/models` | GET | 获取所有模型列表 |
| `/api/models/{model_id}` | GET | 获取指定模型信息 |
| `/api/models/{model_id}/attributes` | GET | 获取模型属性 |
| `/api/models/{model_id}/associations` | GET | 获取模型关联关系 |

### 实例相关接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/models/{model_id}/instances` | GET | 获取模型实例列表（支持分页、搜索、排序） |
| `/api/models/{model_id}/instances/search` | POST | 高级搜索实例 |
| `/api/models/{model_id}/instances` | POST | 创建实例 |
| `/api/models/{model_id}/instances/{instance_id}` | PUT | 更新实例 |

### 关联相关接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/find/associationtype` | POST | 获取关联类型列表 |
| `/find/objectassociation` | POST | 获取模型关联列表 |
| `/find/instassociation` | POST | 获取实例关联列表 |
| `/create/instassociation` | POST | 创建实例关联 |
| `/delete/instassociation/{obj_id}/{inst_asst_id}` | DELETE | 删除实例关联 |

**代码位置**：`/workspace/bk-cmdb/cmdb_server_lite/main.py`

## Lite 与 原项目 的表结构对比

| 原项目 (MongoDB) | Lite 项目 (DuckDB) | 说明 |
|------------------|-------------------|------|
| cc_ObjDes | cc_ObjDes | 模型定义表（完全相同） |
| cc_ObjAttDes | cc_ObjAttDes | 属性定义表（完全相同） |
| cc_AsstDes | cc_AsstDes | 关联类型表（完全相同） |
| cc_ObjAsst | cc_ObjAsst | 模型关联表（完全相同） |
| cc_ObjectBase_{supplier}_pub_{obj} | cc_ObjectBase_0_pub_{obj} | 实例数据表（supplier固定为0） |
| cc_InstAsst_{supplier}_pub | cc_InstAsst_0_pub | 实例关联表（supplier固定为0） |
| cc_ObjClassification | cc_ObjClassification | 模型分类表（保留） |
| cc_PropertyGroup | 保留 | 属性分组表（保留） |

**关键差异**：

1. **供应商账号**：Lite版本固定使用 `bk_supplier_account = '0'`，不支持多租户
2. **表名格式**：实例表命名格式为 `cc_ObjectBase_0_pub_{obj_id}`（原项目为 `cc_ObjectBase_{supplier}_pub_{obj_id}`）
3. **数据库类型**：原项目使用MongoDB（文档型），Lite版本使用DuckDB（关系型）
4. **ID生成**：DuckDB不支持AUTOINCREMENT，需手动管理ID

## 关联数据一致性说明

### 关联类型ID匹配

在创建实例关联时，必须确保以下ID的正确匹配：

| 表 | 字段 | 说明 |
|----|------|------|
| cc_InstAsst_0_pub | bk_obj_asst_id | 必须匹配 cc_ObjAsst.bk_obj_asst_id |
| cc_InstAsst_0_pub | bk_relation_type_id | 必须匹配 cc_AsstDes.bk_asst_id |
| cc_ObjAsst | bk_asst_id | 必须匹配 cc_AsstDes.bk_asst_id |

**示例**（SLB关联场景）：

```sql
-- cc_AsstDes 中的关联类型
SELECT bk_asst_id FROM cc_AsstDes WHERE bk_asst_name = 'SLB服务部署';
-- 返回: slb_to_server

-- cc_ObjAsst 中的模型关联
SELECT bk_obj_asst_id, bk_asst_id FROM cc_ObjAsst WHERE bk_obj_id = 'bk_slb';
-- 返回: bk_slb_server_rel, slb_to_server

-- cc_InstAsst_0_pub 中的实例关联
INSERT INTO cc_InstAsst_0_pub (
    bk_obj_asst_id,        -- 必须为: bk_slb_server_rel
    bk_relation_type_id    -- 必须为: slb_to_server
) VALUES (...);
```

## 注意事项

1. **表名命名**：实例表命名格式为 `cc_ObjectBase_0_pub_{bk_obj_id}`

2. **主键ID**：DuckDB不支持AUTOINCREMENT，需手动获取MAX(id)+1

3. **字段命名**：
   - 使用下划线命名风格（如 `bk_obj_id`）
   - 必须包含系统字段：`id`, `bk_inst_id`, `bk_inst_name`, `bk_obj_id`

4. **关联关系**：
   - `cc_ObjAsst` 定义模型级别的关联
   - `cc_InstAsst_0_pub` 定义实例级别的关联
   - 必须保持 `bk_asst_id` 和 `bk_obj_asst_id` 的一致性

5. **数据迁移**：
   - 从原项目迁移时，需将MongoDB的ObjectId转换为DuckDB的INTEGER ID
   - 供应商账号统一设置为 '0'
