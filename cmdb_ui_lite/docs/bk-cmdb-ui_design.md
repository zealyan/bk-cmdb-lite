# 原UI项目功能菜单文档

本文档基于 bk-cmdb 源码 `src/ui/src/views/` 目录下的路由配置分析整理而成，记录了配置管理数据库（CMDB）前端的完整功能菜单结构。

## 一、菜单总览

bk-cmdb 系统包含以下六大主菜单模块：

| 序号 | 主菜单 | 英文名称 | 路由前缀 | 说明 |
|------|--------|----------|----------|------|
| 1 | 首页 | Index | `/index` | 系统默认首页 |
| 2 | 业务 | Business | `/business/:bizId` | 业务相关的所有功能 |
| 3 | 业务集 | Business Set | `/business-set/:bizSetId` | 业务集管理功能 |
| 4 | 资源 | Resource | `/resource` | 主机、实例等资源管理 |
| 5 | 模型 | Model | `/model` | 模型定义与管理 |
| 6 | 运营分析 | Analysis | `/analysis` | 审计与统计功能 |
| 7 | 平台管理 | Platform Management | `/platform-management` | 全局配置管理 |

---

## 十一、技术说明

### 11.1 路由架构

- **路由模式**：采用 Hash 路由模式（`mode: 'hash'`），便于 SPA 应用在无服务器配置环境下运行
- **路由组织**：采用模块化配置，按功能模块分离路由配置文件
- **动态路由**：使用 Vue Router 的动态路由加载，按需加载页面组件
- **路由守卫**：实现了完整的路由守卫机制，包括权限校验、业务拦截、请求取消等

### 11.2 路由配置文件位置

| 模块 | 配置文件路径 |
|------|-------------|
| 首页 | `views/index/router.config.js` |
| 资源 | `views/resource/router.config.js` |
| 业务 | `views/business/router.config.js` |
| 业务集 | `views/business-set/router.config.js` |
| 模型 | `views/model-manage/router.config.js` |
| 运营分析 | `views/audit/router.config.js` |
| 平台管理 | `views/global-config/router.config.js` |

### 11.3 菜单权限控制

系统采用 IAM（Identity and Access Management）进行权限控制，每个菜单项都配置了对应的权限标识，通过 `meta.auth` 字段进行声明。
