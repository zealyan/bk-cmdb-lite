# 高级筛选条件保持功能测试指南

## 测试目标
验证从实例列表页进入详情页，再通过面包屑返回时，高级筛选条件是否保持不变。

## 修复概述

### 问题
之前的实现中，从详情页返回列表页后，高级筛选条件标签会消失，即使URL中有filter_adv参数。

### 修复的关键代码

#### 1. general-model/index.vue - restoreStateFromUrl 方法
**修复前：**
```javascript
} else {
  this.advancedFilterConditions = null
}

this.updateFilterTagsFromQuery()  // 总是会调用，清空高级筛选标签
```

**修复后：**
```javascript
} else {
  this.advancedFilterConditions = null
  // 没有高级筛选条件时，才使用简单搜索条件更新filterTags
  this.updateFilterTagsFromQuery()
}
```

#### 2. general-model/index.vue - handleViewDetails 方法
确保在进入详情页前，将高级筛选条件保存到URL中：
```javascript
handleViewDetails(instance) {
  // 进入详情页前，确保URL中包含当前的高级筛选条件
  const query = this.$route.query
  const filterAdv = query.filter_adv || (this.advancedFilterConditions ? QS.stringify(
    Object.keys(this.advancedFilterConditions).reduce((acc, id) => {
      const { operator, value } = this.advancedFilterConditions[id]
      const key = `${id}.${operator.replace('$', '')}`
      if (String(value).length) {
        acc[key] = Array.isArray(value) ? value.join(',') : value
      }
      return acc
    }, {}), { encode: false }
  ) : null)
  
  const s = query.s || 'adv'
  
  this.syncStateToUrl({ 
    filter_adv: filterAdv,
    s: filterAdv ? s : undefined
  })
  this.prevInstanceId = instance.id
  this.$router.push({
    name: 'ResourceInstanceDetails',
    params: { objId: this.objId, instId: instance.id }
  })
}
```

## 手动测试步骤

### 步骤 1: 访问实例列表页
1. 打开浏览器，访问 http://localhost:8082
2. 点击导航到实例列表页（例如：交换机实例列表）
3. URL 应该类似：`http://localhost:8082/#/resource/instance/bk_switch`

### 步骤 2: 设置高级筛选条件
1. 点击高级筛选按钮（漏斗图标）
2. 在筛选面板中设置一个或多个筛选条件
3. 点击"搜索"或"确定"按钮提交筛选
4. 验证：
   - URL 中应该包含 `filter_adv` 参数
   - 页面上应该显示筛选标签（filter tags）
   - 列表数据应该按照筛选条件过滤

### 步骤 3: 进入详情页
1. 点击任意一个实例的"查看"按钮或ID链接
2. 进入实例详情页
3. 记录当前 URL（如果有的话，应该保留部分 query 参数）

### 步骤 4: 返回列表页
1. 点击面包屑导航中的返回按钮（箭头图标），或者点击第一个面包屑项
2. 或者使用浏览器的返回按钮

### 步骤 5: 验证筛选条件保持
1. 检查 URL 是否仍然包含 `filter_adv` 参数
2. 检查筛选标签是否仍然显示在页面上
3. 检查列表数据是否仍然按照筛选条件过滤
4. 打开浏览器开发者工具，查看控制台日志，确认：
   - `[restoreStateFromUrl]` 被调用
   - `hasAdvancedFilter` 为 `true`
   - `filterTagsCount` 大于 0

## 预期结果

✅ **成功：**
- 返回后 URL 仍包含 `filter_adv` 参数
- 筛选标签保持显示
- 列表数据保持筛选状态

❌ **失败：**
- URL 中 `filter_adv` 参数消失
- 筛选标签消失
- 列表数据恢复为未筛选状态
